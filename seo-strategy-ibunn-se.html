Key Findings
The iBunn website has zero structured data markup (JSON-LD), zero Open Graph tags, and zero Twitter Card tags — meaning search engines and social platforms have no rich metadata to work with. The meta description is completely missing, which means Google will auto-generate a snippet from page content, reducing your click-through rate. Content is critically thin at approximately 200 words, while top-ranking Swedish IT consulting competitors like Prog-IT (5,000+ words), Castra (2,500+ words), and DevCore (1,800+ words) provide substantially more depth. The title tag “iBunn - IT-konsultföretag” at 25 characters is too short and doesn’t include the location “Stockholm” — a critical keyword all competitors use.

Robots.txt & Sitemap

No robots.txt file

No robots.txt found in the project directory. Search engines won’t have crawling directives and won’t find your sitemap reference. Every production website should have a robots.txt.

Create a robots.txt at the site root with:
User-agent: *
Allow: /
Sitemap: https://ibunn.se/sitemap.xml
High

No sitemap.xml file

No sitemap.xml found. While Google can discover your single page through crawling, a sitemap signals to search engines which URLs are important and when they were last modified.

Create a sitemap.xml listing your page URL with a lastmod date. Even for a single-page site, this is a best practice that signals professionalism to search engines.

Technical Infrastructure

Using Tailwind CSS CDN instead of production build

The page loads Tailwind from cdn.tailwindcss.com. This is a development convenience — in production it delivers the entire Tailwind framework (~300KB+) instead of only the classes you actually use (~10-30KB with purging). This significantly impacts page load speed.

Switch to a production Tailwind build using PostCSS with purging enabled. This can reduce CSS size by 90%+ and improve Core Web Vitals scores.
Medium

No image lazy loading

None of the 6 images use loading="lazy". Service card images below the fold are loaded immediately on page load, increasing initial page weight unnecessarily.

Add loading="lazy" to all images except the hero background. The hero should use loading="eager" or have a preload hint to load first.
Medium

No font preconnect/preload hints

Google Fonts is loaded without preconnect hints. The browser must discover, connect to, and download from fonts.googleapis.com and fonts.gstatic.com sequentially, adding latency to text rendering.

Add before the font stylesheet:
<link rel="preconnect" href="https://fonts.googleapis.com">
<link rel="preconnect" href="https://fonts.gstatic.com" crossorigin>
Low

All images are PNG format

All 6 images are PNG files. Modern formats like WebP or AVIF can reduce file sizes by 25-50% with comparable quality, improving page load speed and Core Web Vitals.

Convert images to WebP format with PNG fallback. Use <picture> elements for format negotiation.
Medium
